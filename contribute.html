Contributing to TRANSPOSE

Help advance travel demand modeling by contributing code, documentation, data, or expertise. We welcome contributors at all skill levels.

Development Setup

Most TDToolkit components are implemented in Python. Here's how to set up your development environment:

Using Conda

# Clone the repository
git clone https://github.com/transpose-org/[component-name].git
cd [component-name]

# Create conda environment
conda env create -f environment.yml
conda activate tdtoolkit

# Install development dependencies
pip install -e ".[dev]"

# Run tests
pytest tests/

Using Poetry

# Clone the repository
git clone https://github.com/transpose-org/[component-name].git
cd [component-name]

# Install dependencies
poetry install

# Activate virtual environment
poetry shell

# Run tests
poetry run pytest

Pre-commit Hooks

We use pre-commit hooks to maintain code quality:

# Install pre-commit hooks
pre-commit install

# Run hooks manually
pre-commit run --all-files

Pull Request Guidelines

We follow these guidelines to maintain code quality and ensure smooth collaboration:

  • Keep PRs focused: One feature or bug fix per PR. Small, scoped changes are easier to review.
  • Write tests: All new features must include unit tests. Bug fixes should include regression tests.
  • Update documentation: Include docstrings, update README if needed, and add user guide entries for new features.
  • Follow style guides: Python code follows PEP 8. Use Black for formatting and flake8 for linting.
  • Write clear commit messages: Use conventional commits format (feat:, fix:, docs:, etc.)

PR Process

  1. Fork the repository and create a feature branch
  2. Make your changes with appropriate tests
  3. Run the test suite locally
  4. Push to your fork and open a PR against the main branch
  5. Address review feedback promptly
  6. Celebrate when merged!

Testing Standards

All components maintain high test coverage:

  • Unit tests for individual functions and classes
  • Integration tests for component interactions
  • Benchmark tests to prevent performance regressions
  • Documentation tests for code examples
# Run full test suite
pytest

# Run with coverage
pytest --cov=tdtoolkit --cov-report=html

# Run specific test file
pytest tests/test_models.py

# Run benchmarks
pytest benchmarks/ --benchmark-only

Code Style

Consistent style across all TDToolkit components:

# Format code with Black
black src/ tests/

# Check style with flake8
flake8 src/ tests/

# Type checking with mypy
mypy src/

# Sort imports
isort src/ tests/
